# FreeRADIUS Manager

A production-ready, web-based FreeRADIUS management platform with Docker deployment, full RBAC, real-time monitoring, and enterprise security.

## Features

| Category | Details |
|---|---|
| **Authentication** | JWT with refresh token rotation, MFA (TOTP), bcrypt password hashing |
| **Roles** | Super Admin, Admin, Operator (granular RBAC) |
| **RADIUS Users** | Create, edit, suspend, delete, bulk import/export CSV |
| **Device Limiting** | Per-user simultaneous session limits (1–20 devices) |
| **NAS Management** | Add/edit/delete NAS clients, test connectivity, auto-discover |
| **EAP Support** | PEAP, EAP-TLS, EAP-TTLS via FreeRADIUS |
| **Monitoring** | Real-time sessions, auth logs, audit trail |
| **Security** | Rate limiting, brute-force protection, CORS, audit logging |
| **Docker** | One-command deployment with `docker-compose up -d` |

## Quick Start

### Prerequisites
- Docker 20.10+
- Docker Compose 2.0+
- 2 GB RAM minimum
- Ports 8081, 8080, 1812/UDP, 1813/UDP available

### Deployment

```bash
# 1. Clone or download this project
cd /path/to/radius-manager

# 2. Run the automated setup script
chmod +x setup.sh && ./setup.sh

# Or manually:
cp .env.example .env
# Edit .env with your settings
docker-compose up -d --build
```

### Access
- **Web UI:** http://your-server:8081
- **Default credentials:** `superadmin` / `Admin@123456`
- **⚠️ Change the password immediately after first login!**

## User Roles

### Super Admin
- Full system access
- Create/manage admin and operator users
- System settings and backup management
- View all audit logs

### Admin  
- Manage RADIUS users (create, edit, delete, suspend)
- Manage NAS clients
- View all monitoring data
- Cannot modify system settings

### Operator
- View RADIUS user list
- Reset user passwords only
- View connection logs
- Cannot create/delete users or modify NAS devices

## Configuring Network Devices

### MikroTik RouterOS
```
/radius
add address=YOUR_SERVER_IP secret=YOUR_SHARED_SECRET service=login,wireless

/ip hotspot profile
set profile1 use-radius=yes
```

### Cisco IOS
```
aaa new-model
radius-server host YOUR_SERVER_IP auth-port 1812 acct-port 1813
radius-server key YOUR_SHARED_SECRET
aaa authentication login default group radius local
```

### Ubiquiti UniFi
Go to **Settings → Profiles → RADIUS** and add:
- IP: your server IP
- Port: 1812
- Secret: your shared secret
- Accounting port: 1813

### pfSense / OPNsense
**Services → FreeRADIUS** or configure under **VPN → Captive Portal → RADIUS Settings**

## Environment Variables

See `.env.example` for all configuration options.

Key variables:
- `DB_PASSWORD` — PostgreSQL password (auto-generated by setup.sh)
- `JWT_SECRET` — JWT signing secret (auto-generated by setup.sh)
- `RADIUS_SECRET` — FreeRADIUS shared secret for backend ↔ FreeRADIUS
- `ADMIN_IP_WHITELIST` — Restrict admin access to specific IPs (e.g., `10.0.0.0/8`)

## API Reference

Base URL: `http://your-server:8080/api/v1`

All endpoints except `/auth/login` and `/health` require:
```
Authorization: Bearer <access_token>
```

### Authentication
| Method | Path | Description |
|--------|------|-------------|
| POST | `/auth/login` | Login with username/password |
| POST | `/auth/refresh` | Refresh access token |
| POST | `/auth/logout` | Revoke refresh token |
| POST | `/auth/change-password` | Change own password |
| POST | `/auth/mfa/setup` | Setup TOTP MFA |
| POST | `/auth/mfa/verify` | Enable MFA |

## Maintenance

### View Logs
```bash
# All services
docker-compose logs -f

# Specific service
docker-compose logs -f freeradius
docker-compose logs -f backend
docker-compose logs -f frontend
docker-compose logs -f postgres
```

### Backup Database
```bash
docker exec radius_postgres pg_dump -U radius_user radius > backup_$(date +%Y%m%d).sql
```

### Restore Database
```bash
cat backup_20240101.sql | docker exec -i radius_postgres psql -U radius_user radius
```

### Update to New Version
```bash
git pull
docker-compose down
docker-compose build --no-cache
docker-compose up -d
```

### Debug RADIUS Authentication
```bash
# Test a RADIUS auth from inside the FreeRADIUS container
docker exec -it radius_freeradius radtest username password 127.0.0.1 0 testing123

# View live RADIUS debug output
docker exec -it radius_freeradius freeradius -X
```

## Security Checklist

Before going to production:
- [ ] Change default `superadmin` password
- [ ] Generate strong DB_PASSWORD, JWT_SECRET, RADIUS_SECRET
- [ ] Set `ADMIN_IP_WHITELIST` to restrict admin access
- [ ] Enable HTTPS (use a reverse proxy like nginx/Traefik with Let's Encrypt)
- [ ] Configure SMTP for email notifications
- [ ] Review NAS clients and ensure strong shared secrets (32+ chars)
- [ ] Enable MFA for all admin users
- [ ] Set appropriate firewall rules (only expose 8081, 1812/UDP, 1813/UDP)

## Troubleshooting

**Can't connect to web interface:**
- `docker-compose ps` — check all containers are running
- `docker-compose logs frontend` — check for errors

**RADIUS authentication failing:**
- Check FreeRADIUS logs: `docker-compose logs freeradius`
- Verify NAS secret matches the device configuration
- Test with radtest: `docker exec -it radius_freeradius radtest user pass localhost 0 secret`

**Database connection error:**
- `docker-compose logs postgres` — check PostgreSQL status
- Verify `DB_PASSWORD` in `.env` matches what PostgreSQL was initialized with

**Cannot log in to web interface:**
- Default credentials: `superadmin` / `Admin@123456`
- Reset via SQL: `docker exec -it radius_postgres psql -U radius_user radius -c "UPDATE app_users SET password_hash='...' WHERE username='superadmin'"`
